### 1. Added a site screenshot to the README

**Why `file://` didn't work:** the Playwright MCP browser blocks the `file:`
protocol, so the page had to be served over HTTP first.

Steps to reproduce:

1. Serve the site locally (Python isn't installed on this machine, so use Node):
   ```powershell
   npx --yes http-server -p 8765 -c-1
   ```
   This serves the current folder at `http://127.0.0.1:8765`.

2. In Playwright MCP: navigate to `http://127.0.0.1:8765/index.html`,
   wait ~3s for the hero video / fade-ins, then take a **full-page** screenshot
   saved as `screenshot.png` in the project root.

3. Reference it in `README.md`, right under the intro paragraph:
   ```markdown
   ![Travel Explorer screenshot](screenshot.png)
   ```

4. Ignore Playwright's scratch artifacts by adding to `.gitignore`:
   ```

   .playwright-mcp/
   ```

5. Commit and push:
   ```powershell
   git add README.md screenshot.png .gitignore
   ```

### 3. Connected the enquiry form to Web3Forms (real email delivery)

**Problem found:** the enquiry form's old `submitEnquiry()` only saved to the
browser's `localStorage` — submissions never reached anyone. The "We'll be in
touch" message was misleading.

**Fix:** rewired `submitEnquiry()` in `index.html` to POST to the Web3Forms API
so enquiries arrive by email.

1. Got a free access key at https://web3forms.com (enter email, key is shown).
   Key in use: `5c6b48a7-374e-4b83-a442-8b8a3c901821`
   (Web3Forms access keys are meant to be public — safe in client-side code.)
2. `submitEnquiry()` is now `async` and does:
   ```js
   await fetch('https://api.web3forms.com/submit', {
     method: 'POST',
     headers: { 'Content-Type': 'application/json', 'Accept': 'application/json' },
     body: JSON.stringify({ access_key: WEB3FORMS_ACCESS_KEY, ...enquiry })
   });
   ```
3. Added a "Sending…" button state, plus success and error message boxes.
4. Enquiries are emailed to the address tied to the Web3Forms account.

**Still to verify:** submit a real enquiry on the live site and confirm the
email arrives. (A live Playwright test couldn't run — the MCP browser kept
dropping its session.)
